什么是 Skill？

一段 Markdown 指令，告诉 Copilot 如何执行特定任务

📊

解析 Media 日志

→ 输出质量报告

📈

分析 CQD 数据

→ 自动脱敏 + 统计

✉️

写邮件

→ 生成 HTML 格式

Skill = Structured Prompt

把你脑子里 "我平时是这样做的" 写成 Copilot 能执行的步骤

❌ 没有 Skill

每次都要告诉 Copilot 怎么做

✅ 有了 Skill

Copilot 自动按你的 SOP 执行

两阶段注入机制

01

描述匹配

读取所有 SKILL.md 的 description

→ 判断哪个 Skill 最匹配

→
02

完整注入

读取整个 SKILL.md 内容

→ 按 Procedure 执行

💡 所以 description 要写好，不然 Skill 匹配不上！

SKILL.md 核心要素

--- description: 解析 Teams mediastack .blog， 输出 CombinedMediaLogs.txt --- # decode-media-log 将 .blog 转为可读文本。 ## Procedure 1. 定位 MediaDecode.exe 路径 2. 执行解码命令 3. 输出到指定目录

description

触发匹配的关键

Procedure

具体执行步骤

Clone + Junction 配置

# 1. Clone Team Skills Repo git clone <team-repo-url> C:\Users\$env:USERNAME\.copilot\Teamskillrepo # 2. 创建 Junction（符号链接） New-Item -ItemType Junction ` -Path "C:\Users\$env:USERNAME\.copilot\skills\team" ` -Target "C:\Users\$env:USERNAME\.copilot\Teamskillrepo"

Team Skills 同步到 ~/.copilot/skills/team/

💡 Git pull 即可获取最新 Skills

创建自己的 Skill

最简单的方式：

1 先让 Copilot 帮你解决一个问题
2
解决后说： "把刚才的解决步骤总结成一个 Skill"
3 Copilot 自动生成 SKILL.md

💡 参考：github.com/anthropics/agent-skills (109k+ ⭐)

文件结构

~/.copilot/skills/ ├── my-skill/ │ ├── SKILL.md # 主入口（必须） │ ├── references/ # 参考文档 │ │ └── playbook.md │ ├── scripts/ # 脚本 │ │ └── helper.ps1 │ └── assets/ # 模板/资源 └── team -> Teamskillrepo # Junction
💡

SKILL.md 是唯一必须的文件
